# Guía de Mejores Prácticas Frontend Moderno (2026)

Este skill proporciona directrices para construir aplicaciones web modernas, eficientes y de alta calidad, integrando las mejores prácticas de la industria y el uso de agentes de IA.

## 1. Estándares Técnicos Core

- **TypeScript por defecto**: El uso de JavaScript puro se considera legado para proyectos profesionales. TypeScript proporciona seguridad de tipos de extremo a extremo.
- **Meta-Frameworks**: Preferir frameworks como **Next.js**, **Nuxt** o **Vite** con React/Vue para manejar enrutamiento, renderizado y optimización de forma nativa.
- **Desarrollo Basado en Componentes**: Construir interfaces como asambleas de componentes pequeños, aislados y reutilizables.

## 2. Arquitectura y Rendimiento

- **Edge-First Development**: Mover la lógica lo más cerca posible del usuario para reducir la latencia.
- **Core Web Vitals**: Monitorización continua de métricas de rendimiento (LCP, FID, CLS).
- **Optimización Automática**: Uso de compiladores inteligentes para memoización y bundling eficiente.

## 3. Calidad y Accesibilidad (A11y)

- **Accesibilidad desde el inicio**: Implementar estándares WCAG usando HTML semántico y pruebas automatizadas.
- **Pruebas Automatizadas**: 
    - **Unitarias**: Para lógica de negocio y utilidades.
    - **Integración**: Para flujos de componentes.
    - **E2E**: Para flujos críticos de usuario (Playwright/Cypress).

## 4. Estética y Diseño Premium

- **Tipografía Distintiva**: Evitar fuentes genéricas (Inter, Roboto). Optar por combinaciones que eleven la estética.
- **Diseño Dinámico**: Implementar micro-animaciones, gradientes suaves y efectos de profundidad (glassmorphism).
- **Consistencia Visual**: Uso rigoroso de variables CSS y sistemas de diseño.

## 5. Flujos de Trabajo con Agentes de IA

- **IA como Arquitecto**: Utilizar agentes para andamiar (scaffold) características, trazar flujos de datos y generar documentación.
- **Integración de Skills**: Potenciar el flujo de trabajo con skills de [Skills.sh](https://skills.sh) como `frontend-design` y `frontend-testing`.
